## 2025-05-15 - Improving Tooltip Accessibility and Focus Visibility
**Learning:** In the CyberUI design system, many interactive elements used `focus:outline-none` without providing alternative focus indicators, making them inaccessible to keyboard users. Additionally, manual CSS tooltips were being used alongside standard Radix UI tooltips, causing redundancy and poor semantics (e.g., nesting block-level `div`s inside buttons).
**Action:** Always replace `focus:outline-none` with `focus-visible:ring-2` (or similar) to ensure focus visibility. When using Radix UI `TooltipTrigger`, ensure the trigger is a semantic element like a `button` (with `type="button"`) and its children are phrasing content (like `span`). Always remove redundant manual tooltip implementations in favor of standardized components.
